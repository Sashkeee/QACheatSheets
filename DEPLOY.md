# Инструкция по деплою QA CheatSheet на VPS

Чтобы запустить этот сайт рядом с **OrangeBrew**, мы будем использовать Docker и обновим существующий **Caddyfile**.

## 1. Сборка и запуск контейнера

В папке `app` уже созданы `Dockerfile` и `docker-compose.yml`.

1. Перенесите папку `app` на сервер (или выполните `git pull`, если используете репозиторий).
2. Запустите проект:
   ```bash
   docker compose up -d --build
   ```
   *Сайт будет доступен внутри сервера на порту **8082**.*

## 2. Автоматический деплой (CI/CD)

Я настроил GitHub Actions. Теперь при пуше в ветку `main` сайт будет обновляться сам.

### Что нужно сделать ОДИН раз:
Зайди в настройки твоего репозитория на GitHub: **Settings -> Secrets and variables -> Actions** и добавь секреты для твоего VPS:
1.  `VPS_IP` — IP-адрес твоего сервера.
2.  `VPS_USERNAME` — логин (например, `root` или `user`).
3.  `VPS_SSH_KEY` — приватный SSH-ключ для доступа к серверу.

## 3. Настройка домена (Caddy)

Откройте ваш `Caddyfile` (обычно находится в корне `OrangeBrew` на сервере) и добавьте в конец новый блок для поддомена (например, `qa.orangebrew.ru`):

```caddy
qa.orangebrew.ru {
    # Проксируем на контейнер с QA CheatSheet
    reverse_proxy localhost:8082
}
```

*Если вы хотите использовать другой поддомен, просто замените `qa.orangebrew.ru` на ваш вариант.*

## 3. Перезапуск Caddy

Чтобы изменения вступили в силу, перезагрузите Caddy:
```bash
docker exec -it orangebrew-caddy caddy reload --config /etc/caddy/Caddyfile
```
*(Имя контейнера может отличаться, уточните через `docker ps`)*

---

### Почему это безопасно для OrangeBrew?
* **Изоляция**: QA CheatSheet работает в своем контейнере и на своем порту (8082), не пересекаясь с 3000/8080 портами OrangeBrew.
* **Единый вход**: Caddy просто распределяет трафик: если запрос идет на `orangebrew.ru` — он летит в старый проект, если на `qa.orangebrew.ru` — в новый.
