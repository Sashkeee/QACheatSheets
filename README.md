# 🚀 QA CheatSheet — Интерактивный справочник инженера по тестированию

**QA CheatSheet** — это современное веб-приложение, представляющее собой глубокую базу знаний для начинающих и практикующих QA-специалистов. Проект создан не просто как «выжимка» терминов, а как полноценный образовательный ресурс с реальными примерами, аналогиями из профильной литературы и визуальными схемами.

---

## 📚 Основные модули и темы

Справочник охватывает все ключевые области современного тестирования:

*   **🧪 Основы тестирования**: Жизненный цикл разработки (SDLC), концепция Testbase, пирамида тестирования и цена ошибки на разных этапах.
*   **🎨 Тест-дизайн**: Техники «Черного ящика» (Эквивалентное разделение, Граничные значения) на примере сказки про Золушку и других понятных аналогий. Артефакты: от чек-листов до сложных баг-репортов.
*   **📊 Базы данных (SQL)**: Практическое руководство по SELECT, типам JOIN, агрегатным функциям и подзапросам на базе классической схемы Sakila.
*   **🌐 API и Сети**: Архитектура REST, методы HTTP (GET, POST, PUT, DELETE), коды ответов и работа с инструментами типа Postman.
*   **🤖 Автоматизация**: Понимание пирамиды автоматизации, паттерн Page Object Model (POM) и основы написания автотестов.
*   **🛡️ Безопасность**: Базовые уязвимости из OWASP Top 10 (SQL Injection, XSS, IDOR) с примерами их поиска.
*   **💼 Карьера и Инфраструктура**: Подготовка резюме, локализация багов, работа с Git и Docker для тестировщиков.
*   **🧠 AI в тестировании**: Как использовать нейросети для генерации тестовых данных, анализа логов и написания кода.

## 🛠 Технологический стек

Проект построен на современных инструментах для обеспечения скорости и стабильности:

*   **Frontend**: React 19, TypeScript, Vite.
*   **Styling**: Tailwind CSS (современный адаптивный дизайн с поддержкой Dark Mode).
*   **Icons & Charts**: Lucide React, Recharts.
*   **Infrastructure**: Docker, Docker Compose.
*   **CI/CD**: GitHub Actions (автоматический деплой при пуше в `main`).
*   **Reverse Proxy**: Caddy (с автоматическим получением SSL/HTTPS).

## ⚙️ Запуск локально

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/Sashkeee/QACheatSheets.git
    cd QACheatSheets/app
    ```
2.  Установите зависимости:
    ```bash
    npm install --legacy-peer-deps
    ```
3.  Запустите сервер для разработки:
    ```bash
    npm run dev
    ```
    *Сайт будет доступен по адресу `http://localhost:5173`.*

## 🐋 Запуск через Docker

Если вы хотите запустить проект максимально близко к серверной среде:
```bash
cd app
docker compose up -d --build
```
*Приложение будет доступно на порту `8082` вашего компьютера.*

## 🚀 Деплой и CI/CD

В проекте настроен автоматический пайплайн:

1.  Каждый пуш в ветку `main` запускает сборку Docker-образа в **GitHub Actions**.
2.  Собранный образ отправляется в **GitHub Container Registry (GHCR)**.
3.  GitHub автоматически подключается к вашему VPS по SSH и обновляет контейнер.

Для работы деплоя необходимо настроить **Secrets** в репозитории:
*   `VPS_IP` — IP-адрес сервера.
*   `VPS_USERNAME` — логин пользователя (например, `root`).
*   `VPS_SSH_KEY` — приватный SSH-ключ.
